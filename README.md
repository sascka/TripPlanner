# TripPlanner

TripPlanner - это мой проект для планирования поездок. Идея простая: перед поездкой часто надо держать в голове билеты, бронь, список дел, заметки и погоду. Я сделал сайт, где всё это можно собрать в одном месте.

## Что умеет проект

- регистрация и вход в аккаунт;
- создание поездок;
- редактирование и удаление поездок;
- чек-лист подготовки;
- заметки внутри поездки;
- загрузка документов в формате PDF, PNG, JPG и JPEG;
- просмотр погоды через OpenWeatherMap;
- REST API для части данных;
- генерация PDF-отчёта;
- ссылка общего доступа для просмотра поездки;
- светлая и тёмная тема.

## Как запустить

Сначала надо перейти в папку проекта и установить зависимости:

```bash
py -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

Потом нужно создать файл `.env`. Пример есть в `.env.example`.

```env
SECRET_KEY=change-me
OPENWEATHER_API_KEY=ваш_ключ
DATABASE_URL=sqlite:///instance/tripplanner.sqlite3
```

После этого можно запускать:

```bash
py main.py
```

Сайт открывается по адресу:

```text
http://127.0.0.1:5000
```

## Погода

Для погоды используется OpenWeatherMap. Ключ записывается в `.env`:

```env
OPENWEATHER_API_KEY=ваш_ключ
```

Если ключ не указан, сайт всё равно работает, просто вместо погоды показывает сообщение.

## База данных

В проекте используется SQLite. Файл базы создаётся здесь:

```text
instance/tripplanner.sqlite3
```

В базе хранятся пользователи, поездки, чек-листы, заметки и информация о файлах. Сами файлы лежат отдельно в папке `uploads`.

## API

В проекте есть несколько API-адресов:

- `GET /api/trips` - список поездок пользователя;
- `GET /api/trips/<id>` - подробная информация о поездке;
- `POST /api/trips/<id>/checklist` - добавить пункт чек-листа;
- `PATCH /api/checklist/<id>` - поменять статус пункта;
- `DELETE /api/checklist/<id>` - удалить пункт;
- `POST /api/trips/<id>/notes` - добавить заметку;
- `DELETE /api/notes/<id>` - удалить заметку;
- `GET /api/weather?city=Казань` - получить погоду.

Лучше всего показать такой сценарий:

1. Зарегистрироваться.
2. Создать поездку.
3. Добавить несколько пунктов чек-листа.
4. Отметить один пункт выполненным.
5. Добавить заметку.
6. Загрузить документ.
7. Показать погоду.
8. Скачать PDF.
9. Открыть ссылку общего доступа.

## Хостинг

Проект подготовлен для запуска через `gunicorn`. Для этого есть `Procfile`.

```bash
gunicorn main:app
```

На хостинге нужно будет задать переменные `SECRET_KEY`, `OPENWEATHER_API_KEY` и при необходимости `DATABASE_URL`.
