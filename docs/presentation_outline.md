# План презентации

## Слайд 1. Название

TripPlanner — сервис «Умный путешественник».

Авторы: указать ФИО участников.

## Слайд 2. Проблема

При подготовке поездки данные находятся в разных местах: билеты, брони, заметки, список дел и прогноз погоды.

## Слайд 3. Идея

Собрать всю информацию о поездке на одной странице: маршрут, даты, бюджет, чек-лист, заметки, файлы и погоду.

## Слайд 4. Основные функции

- регистрация и авторизация;
- создание поездок;
- чек-лист подготовки;
- заметки;
- загрузка документов;
- погода;
- PDF-отчёт;
- общий доступ.

## Слайд 5. Структура проекта

Показать модули `app/models.py`, `app/routes.py`, `app/api.py`, `app/weather.py`, `app/pdf.py`.

## Слайд 6. База данных

Показать модели `User`, `Trip`, `ChecklistItem`, `Note`, `Document` и связи между ними.

## Слайд 7. Использованные технологии

Flask, SQLAlchemy, Flask-Login, SQLite, Bootstrap, JavaScript, REST API, OpenWeatherMap, ReportLab.

## Слайд 8. Демонстрация

1. Зарегистрироваться.
2. Создать поездку.
3. Добавить чек-лист.
4. Загрузить документ.
5. Показать погоду.
6. Скачать PDF.
7. Открыть ссылку общего доступа.

## Слайд 9. Сложности

Рассказать про проверку прав доступа, хранение файлов и получение данных из стороннего API.

## Слайд 10. Итоги и развитие

Проект работает как личный кабинет путешественника. Дальше можно добавить карты, календарь по дням, совместное редактирование и уведомления.
