# Техническое задание

## Название

TripPlanner — сервис «Умный путешественник».

## Цель проекта

Создать веб-приложение, которое помогает пользователю подготовиться к поездке: сохранить маршрут, даты, бюджет, чек-лист, заметки, документы и актуальную погоду.

## Пользователи

Основной пользователь — путешественник, который планирует личную поездку и хочет хранить важные данные в одном месте.

## Основные сценарии

1. Пользователь регистрируется и входит в аккаунт.
2. Пользователь создаёт поездку: название, город, даты, бюджет, описание.
3. Пользователь открывает страницу поездки.
4. Пользователь добавляет пункты чек-листа и отмечает их выполненными.
5. Пользователь добавляет заметки.
6. Пользователь загружает документы: билеты, брони, сканы.
7. Пользователь смотрит погоду для города поездки.
8. Пользователь скачивает PDF-отчёт.
9. Пользователь отправляет другу ссылку для просмотра поездки.

## Функциональные требования

- регистрация и авторизация;
- CRUD для поездок;
- добавление и удаление пунктов чек-листа;
- отметка пунктов чек-листа;
- добавление и удаление заметок;
- загрузка и скачивание файлов;
- генерация PDF-отчёта;
- публичный read-only просмотр поездки по токену;
- REST API для данных приложения;
- интеграция со сторонним API погоды.

## Нефункциональные требования

- приложение должно запускаться локально;
- зависимости должны быть перечислены в `requirements.txt`;
- данные должны храниться в SQLite через ORM;
- код должен быть разделён на модули;
- интерфейс должен быть сделан с Bootstrap или аналогичным фреймворком;
- проект должен быть размещён в git-репозитории.

## Используемые технологии

- Python;
- Flask;
- Flask-SQLAlchemy;
- Flask-Login;
- SQLite;
- Bootstrap;
- OpenWeatherMap API;
- ReportLab;
- HTML, CSS, JavaScript.

## Критерии готовности

- можно создать аккаунт;
- можно создать поездку;
- можно добавить чек-лист, заметки и файлы;
- можно открыть погоду;
- можно скачать PDF;
- можно открыть ссылку общего доступа;
- проект запускается после установки зависимостей.
