# Техническое задание

## Название проекта

TripPlanner.

## Короткое описание

Я делаю веб-приложение для планирования поездок. Пользователь сможет создать поездку, записать важные данные, добавить чек-лист, заметки, документы и посмотреть погоду.

## Зачем нужен проект

Когда человек готовится к поездке, информация часто лежит в разных местах. Например, билеты могут быть в почте, бронь отеля в телефоне, список дел в заметках, а погоду надо смотреть отдельно. Я хочу сделать страницу поездки, где всё это хранится вместе.

## Кто будет пользоваться

Основной пользователь - человек, который планирует поездку и хочет заранее собрать все важные данные.

## Что должно быть в приложении

Пользователь должен иметь возможность:

- зарегистрироваться;
- войти в аккаунт;
- выйти из аккаунта;
- создать поездку;
- изменить поездку;
- удалить поездку;
- посмотреть список своих поездок;
- найти поездку по названию или городу;
- добавить пункты чек-листа;
- отмечать пункты выполненными;
- добавлять заметки;
- загружать документы;
- скачивать документы обратно;
- удалять документы;
- смотреть погоду в городе поездки;
- скачать PDF-отчёт;
- открыть поездку по ссылке общего доступа.

## Данные

В проекте нужны такие сущности:

- пользователь;
- поездка;
- пункт чек-листа;
- заметка;
- документ.

У пользователя может быть несколько поездок. У поездки может быть несколько заметок, документов и пунктов чек-листа.

## Технологии

В проекте используются:

- Python;
- Flask;
- Flask-SQLAlchemy;
- Flask-Login;
- SQLite;
- Bootstrap;
- JavaScript;
- OpenWeatherMap API;
- ReportLab;
- HTML и CSS.

## Что считается готовым проектом

Проект можно считать готовым, если:

- сайт запускается локально;
- можно создать аккаунт;
- можно создать поездку;
- работают чек-листы и заметки;
- можно загрузить файл;
- работает погода при наличии API-ключа;
- можно скачать PDF;
- есть ссылка общего доступа;
- код лежит в git-репозитории;
- есть `requirements.txt`;
- есть пояснительная записка и план презентации.
