# Пояснительная записка

## Идея

Мой проект называется TripPlanner. Это сайт для подготовки поездок. Я выбрал такую тему, потому что перед поездкой обычно появляется много мелких задач: купить билеты, проверить бронь, не забыть документы, записать адрес отеля, посмотреть погоду. Если всё хранить в разных местах, легко что-то потерять.

В TripPlanner каждая поездка открывается как отдельная страница. На ней можно хранить основную информацию, чек-лист, заметки, документы и погоду.

## Что получилось сделать

В проекте есть регистрация и вход в аккаунт. После входа пользователь видит свои поездки. Можно создать новую поездку, указать город, даты, бюджет и описание.

На странице поездки есть несколько частей:

- описание поездки;
- чек-лист подготовки;
- заметки;
- документы;
- погода;
- ссылка общего доступа;
- кнопка для PDF-отчёта.

Также я добавил светлую и тёмную тему. Выбор темы сохраняется в браузере.

## Структура проекта

Я разделил проект на несколько файлов, чтобы код не был собран в одном месте.

`main.py` запускает приложение.

`app/__init__.py` создаёт Flask-приложение, подключает базу данных, авторизацию и маршруты.

`app/config.py` хранит настройки проекта: путь к базе, папку загрузок, ключ погоды и другие параметры.

`app/models.py` содержит модели базы данных.

`app/routes.py` отвечает за обычные страницы сайта и обработку форм.

`app/api.py` отвечает за REST API.

`app/weather.py` получает погоду через OpenWeatherMap.

`app/pdf.py` создаёт PDF-отчёт.

`templates` хранит HTML-шаблоны.

`static` хранит стили и JavaScript.

## База данных

Для хранения данных я использовал SQLite и SQLAlchemy ORM.

В проекте есть такие модели:

- `User` - пользователь;
- `Trip` - поездка;
- `ChecklistItem` - пункт чек-листа;
- `Note` - заметка;
- `Document` - загруженный документ.

Связи сделаны так:

```text
User -> Trip
Trip -> ChecklistItem
Trip -> Note
Trip -> Document
```

Это значит, что один пользователь может создать много поездок, а у каждой поездки могут быть свои чек-листы, заметки и файлы.

Пароли не хранятся обычным текстом. При регистрации пароль превращается в хеш, и уже он записывается в базу.

## Загрузка файлов

К поездке можно прикрепить документы. Разрешены PDF, PNG, JPG и JPEG. Файлы сохраняются в папку `uploads`, а в базе хранится только информация о файле: его имя, тип и к какой поездке он относится.

При загрузке файл получает случайное имя. Это нужно, чтобы файлы с одинаковыми названиями не перезаписывали друг друга.

## Погода

Для погоды используется OpenWeatherMap. Город берётся из поездки. Если API-ключ указан в `.env`, приложение получает температуру, описание погоды и скорость ветра.

Если ключа нет или сервис не отвечает, сайт не падает, а показывает сообщение.

## REST API

Я добавил REST API, чтобы часть данных можно было получать в формате JSON.

Например:

- `/api/trips` возвращает поездки пользователя;
- `/api/trips/<id>` возвращает одну поездку подробнее;
- `/api/weather?city=Казань` возвращает погоду;
- через API можно добавлять и удалять пункты чек-листа и заметки.

На странице поездки JavaScript использует API для обновления погоды без перезагрузки страницы.

## PDF-отчёт

Одна из дополнительных функций - PDF-отчёт. Пользователь может скачать файл с данными поездки. В отчёт попадают даты, город, бюджет, описание, чек-лист, заметки, документы и погода.

PDF создаётся через библиотеку ReportLab.

## Общий доступ

У каждой поездки есть специальная ссылка. По ней другой человек может открыть поездку без входа в аккаунт. Редактировать данные по этой ссылке нельзя, только смотреть.

## Что было сложнее всего

Сложнее всего было правильно связать поездки с пользователями и не дать открыть чужие данные. Для этого перед действиями с поездкой проверяется владелец.

Ещё пришлось отдельно разобраться с загрузкой файлов, потому что файл надо не только сохранить, но и проверить расширение, придумать безопасное имя и записать данные в базу.

## Что можно улучшить

В будущем можно добавить карту маршрута, календарь поездки по дням, совместное редактирование поездки и уведомления о задачах.

## Вывод

В результате получился рабочий веб-сервис для планирования поездок. В проекте используются основные технологии из Web-модуля: Flask, шаблоны, база данных, ORM, авторизация, загрузка файлов, REST API, сторонний API и генерация PDF.
