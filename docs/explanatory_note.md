# Пояснительная записка

## Идея проекта

TripPlanner — веб-сервис для подготовки поездок. Во время планирования путешествия информация часто хранится в разных местах: билеты в почте, брони в мессенджерах, список дел в заметках, прогноз погоды в отдельном приложении. Проект объединяет эти данные на одной странице.

## Назначение

Сервис нужен для удобной подготовки к поездке. Пользователь может заранее составить список дел, загрузить важные документы, сохранить заметки и проверить погоду в городе назначения.

## Структура приложения

Проект разделён на модули:

- `app/__init__.py` создаёт Flask-приложение и подключает расширения;
- `app/models.py` содержит ORM-модели;
- `app/routes.py` содержит HTML-страницы и обработку форм;
- `app/api.py` содержит REST API;
- `app/weather.py` отвечает за запросы к OpenWeatherMap;
- `app/pdf.py` создаёт PDF-отчёт;
- `templates/` содержит HTML-шаблоны;
- `static/` содержит CSS и JavaScript.

## Модели данных

В проекте используются следующие модели:

- `User` — пользователь приложения;
- `Trip` — поездка пользователя;
- `ChecklistItem` — пункт чек-листа подготовки;
- `Note` — заметка по поездке;
- `Document` — загруженный файл.

Связи между моделями построены через SQLAlchemy. Один пользователь может иметь много поездок. У одной поездки может быть много пунктов чек-листа, заметок и документов.

## Реализованные технологии

Проект использует `requirements.txt` для зависимостей, Flask для веб-сервера, SQLAlchemy ORM для работы с базой данных, Flask-Login для авторизации, SQLite для хранения данных, Bootstrap для интерфейса, загрузку файлов через HTML-формы, REST API и сторонний API OpenWeatherMap.

## Оригинальная часть

В проект добавлены две функции, которые выходят за рамки базового CRUD:

- генерация PDF-отчёта по поездке;
- ссылка общего доступа, по которой друг может посмотреть поездку без входа в аккаунт.

## Возможности развития

В дальнейшем можно добавить карту маршрута, совместное редактирование поездок, календарь по дням, уведомления о важных задачах, экспорт в Google Calendar и автоматическое распознавание данных из загруженных билетов.

## Вывод

В результате получился веб-сервис, который решает практическую задачу и использует основные технологии модуля Web: серверные маршруты, шаблоны, базу данных, ORM, авторизацию, файлы, API и хостинг.
